# TICKET-IMP-09 — Horizontal sub-stepper for the CSV import mapper

- **Area:** Import (mapping wizard)
- **Type:** Feature
- **Traceability:** FR-IMP-3 (further refines the mapping-wizard requirement, building on TICKET-IMP-07's guided-feedback work)

## User story

As a user importing a bank CSV, I want the mapper's guided flow to run horizontally — a compact step tracker across the top with one field open at a time below it, previews sitting together up front — instead of scrolling down a tall stack of expanding/collapsing rows, so mapping a file feels like a short wizard rather than a long form.

## Description

TICKET-IMP-07 replaced the mapper's flat simultaneous field list with a guided vertical flow: one field expands, completed fields collapse into a clickable summary row, and the whole thing stacks top-to-bottom. This ticket keeps that guided, one-field-at-a-time spirit but turns it horizontal — a daisyUI step tracker (matching the outer wizard's own Select/Map/Summary stepper) plus a single active-field panel below it — and simplifies the field list itself: the three separate amount/debit/credit fields become one Amount step with a mode toggle, and the two counterparty fields become one Counterparty step, ending in a new Summary step.

## Desired result (to-be)

- Step 2's page becomes, top to bottom: back/confirm buttons (unchanged, outer wizard) → global options (always visible, unchanged fields, no longer split beside a preview) → raw file preview and row preview rendered **side by side** in one row → a horizontal step tracker (reusing the exact daisyUI `steps`/`step`/`step-primary` markup already used at the outer wizard level, [import-wizard.component.html:6-10](../../../src/app/feature-import/components/import-wizard/import-wizard.component.html#L6-L10)) → a single active-field panel showing only the current step's editor.
- The 9 flat fields consolidate into 7 guided steps:
  1. **Date** (`date`, required)
  2. **Description** (`description`, required)
  3. **Amount** — one step with a segmented "Single column" / "Separate debit & credit" toggle (mirroring the actual shipped TICKET-TXN-08 segmented-toggle markup — a `<div class="join">` with two raw `<button class="btn btn-sm join-item">` toggling `btn-primary`/`btn-outline`, see [transaction-filters.component.html:32-51](../../../src/app/feature-transactions/components/transaction-filters/transaction-filters.component.html#L32-L51)) switching between the `amount` select or the `debit`+`credit` selects
  4. **Counterparty** — one step showing both `counterpartyName` and `counterpartyIban` selects together
  5. **Own IBAN** (`ownIban`, optional)
  6. **Balance** (`balance`, optional)
  7. **Summary** — a new final step recapping every mapped field (label + chosen column + resolved sample, reusing the "mapped" text pattern from [column-map-collapsed-field.component.html:12-19](../../../src/app/feature-import/components/column-map-collapsed-field/column-map-collapsed-field.component.html#L12-L19)) plus hosting **both** the "remember this mapping" checkbox and (when offered) the "apply to remaining files" checkbox.
- `ImportMapStepComponent` becomes the sole owner of all of step 2's inner layout. The wizard's parse-state display signals (`parsedRows`, `parseError`, `headerMismatchMessage`, `parsing`, `parseWarnings`) get pushed down as new `input()`s; the wizard's step-2 template shrinks to conditionally rendering `<app-import-map-step>` (or the existing batch-mapping placeholder) with these new inputs — no more `xl:grid-cols-12` split at the wizard level. The actual worker-based debounced parsing stays wizard-owned (it's wizard-lifecycle logic tied to `runCommit()`/batch auto-commit); only the resulting read-only display values move down.
- "Apply to remaining files" state and gating (`applyToRemaining`, `canOfferApplyToRemaining`, `remainingFilesCount`) **stay wizard-owned** — only the checkbox's markup moves into the mapper's Summary step, via new `canOfferApplyToRemaining = input(false)`, `remainingFilesCount = input(0)`, and a new `applyToRemaining = model(false)` bound the same way `[(result)]` already is today.
- `activeFieldKey` (nullable) is replaced by `activeStepId: signal<MapperStepId>` that is **never null** — advancing from Balance lands on `'summary'` as the natural terminus, removing the old "null means guided flow ended" special case.
- Clicking a step in the tracker (`openStep(id)`) only acts for **done-or-current** steps — no jumping ahead to an upcoming step, matching the outer wizard's own stepper behavior. This is a deliberate tightening versus today's vertical layout, where any collapsed row (including ones "ahead," since none technically were) was clickable.
- Amount-mode toggle: `amountMode = signal<'single' | 'split'>('single')`, defaulted right after `detectAndPrefill()`'s patch resolves (`debit || credit` truthy → `'split'`). Switching modes **clears the inactive control(s)** (`debit`/`credit` cleared when switching to single; `amount` cleared when switching to split) — `core/import/csv-row-mapper.ts`'s `resolveAmount` (lines 88-113) gives `mapping.amount` priority whenever truthy, so a stale `debit`/`credit` left behind after switching back to "Single column" would silently persist into a saved/reused `MappingProfile` without the UI ever having shown it was set.
- `resolvedSamples`, `duplicateWarnings`, and `invalidFieldLabels` keep operating at the existing 9-form-control granularity (unchanged logic) — they're simply re-attached to whichever step now groups each control, not rewritten.
- Components superseded/replaced: `column-map-field`, `column-map-active-field`, `column-map-collapsed-field` are removed (the one-active-panel model no longer needs a per-row active/collapsed switch or a stacked list). New components: `column-map-stepper` (horizontal daisyUI steps band), `column-map-simple-field` (single-select editor for Date/Description/Own IBAN/Balance), `column-map-amount-field` (segmented toggle + one or two selects), `column-map-counterparty-field` (both counterparty selects together), `column-map-summary-step` (recap list + both checkboxes), `column-map-sample-caption` (the resolved-sample/duplicate-warning caption pair, shared by the three field editors above once it was duplicated 1×/2×/3× across them). The active step's `@switch` lives directly in `import-map-step.component.html` rather than a separate `column-map-step-panel` router — a dedicated router component would only forward each step's `FormControl`s through with no isolation benefit (unlike the original active/collapsed split, which separated two genuinely complex templates), so it was folded into the template that already owns every control.
- No Dexie schema changes — `MappingProfileColumns` ([core/data-access/app-db.ts:164-175](../../../src/app/core/data-access/app-db.ts#L164-L175)) is unchanged; the amount-mode toggle is transient UI state only. No new `shared/ui` primitive is introduced — `FieldsetComponent`, `SelectComponent`, `ButtonComponent`, `AlertComponent`, `TypographyComponent`, `DividerComponent`, `LabelComponent`, `TableComponent`, `FlexComponent` cover everything needed.

## Acceptance criteria

- [x] The Amount step shows a segmented "Single column"/"Separate debit & credit" toggle; selecting "Single column" shows only the `amount` select, selecting "Separate debit & credit" shows the `debit` and `credit` selects; switching modes clears the now-inactive control(s) so no stale value survives into the persisted `MappingProfile`.
- [x] Amount mode defaults from whatever a bank-preset or saved-profile prefill populated (`split` if `debit`/`credit` were set, `single` otherwise).
- [x] The Counterparty step shows both `counterpartyName` and `counterpartyIban` selects together, both remaining optional.
- [x] The Summary step lists every mapped field (label, chosen column, resolved sample) and hosts the "remember this mapping for this account" checkbox. ~~Superseded same-day~~: the "apply this mapping to the remaining N files" checkbox moved to the always-visible global options section instead (see Notes) — visible whenever `canOfferApplyToRemaining()` is true, not gated on reaching Summary.
- [x] Raw file preview and row (parsed) preview render side by side in one row, both sourced through `ImportMapStepComponent` (row preview via new `parsedRows`/`parseError`/`headerMismatchMessage`/`parsing`/`parseWarnings` inputs) rather than the wizard's previous `xl:grid-cols-12` split; the wizard's own parsing pipeline (debounce, worker call, batch auto-commit `effect`) is untouched.

- [x] `resolvedSamples`, `duplicateWarnings`, and `invalidFieldLabels` continue to operate per individual form control (all 9 keys) and produce identical results to before, just surfaced through the new step groupings.
