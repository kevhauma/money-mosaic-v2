Features Top-level product areas — one node per feature-* folder.
Core Foundation services — repositories, state, stats — one node per subfolder.

Scope rules

  • Feature internals aren't expanded — an edge into a feature node means something in it imports the target, not necessarily its entry point.
  • Core/shared stop one folder past their own root, so e.g. core/data-access/* collapses to data-access.
  • *.spec.ts and node_modules are filtered out entirely.
